# Plan hasta septiembre para empezar DAM

Este plan sirve como referencia para orientar el aprendizaje de Enric antes de comenzar el FP de DAM en septiembre.

## Objetivo principal

Llegar a septiembre con una base solida de logica de programacion y sin miedo a Java.

No se busca aprender muchas tecnologias de forma superficial. Se busca consolidar fundamentos.

## Prioridad actual

Antes de saltar a Java, cerrar bien la calculadora de masa de pizza V1 en Python.

Motivo:

- Enric esta empezando a pasar de ejercicios sueltos a programas reales.
- Python permite consolidar funciones, inputs, diccionarios, calculos y flujo de programa sin demasiada carga sintactica.
- Si se salta demasiado pronto a Java, podria arrastrar dudas basicas a un lenguaje mas estricto.

## Fase 1: cierre de Python aplicado

Objetivo:

Crear una primera version funcional y entendida de la calculadora de masa de pizza.

Debe incluir:

- Funciones pequenas.
- Inputs de usuario.
- Diccionarios.
- Calculos de harina, agua y sal.
- Salida clara por pantalla.
- Codigo limpio.
- Commit y push a GitHub.
- README basico del proyecto cuando toque.

Cuando esta fase este suficientemente cerrada, el profesor debe avisar:

```text
Ahora toca empezar Java como preparacion para DAM.
```

## Fase 2: julio - entrada progresiva a Java

Distribucion recomendada:

- 60% Java.
- 25% Python.
- 15% Git/GitHub y documentacion.

Java:

- Variables.
- Tipos de datos.
- Operadores.
- Condiciones.
- Bucles.
- Metodos.
- Arrays.
- Entrada por teclado con `Scanner`.

Python:

- Mantener la calculadora de pizza.
- Hacer mejoras pequenas.
- Seguir reforzando logica con ejercicios cortos.

Git/GitHub:

- `add`.
- `commit`.
- `push`.
- `pull`.
- Mensajes claros de commit.

## Fase 3: agosto - Java + SQL

Distribucion recomendada:

- 60% Java.
- 25% SQL.
- 15% Python/proyectos.

Java:

- `ArrayList`.
- Clases.
- Objetos.
- Constructores.
- Encapsulacion.
- Programas de consola con menu.
- Ficheros basicos si hay tiempo.

SQL:

- `SELECT`.
- `WHERE`.
- `ORDER BY`.
- `INSERT`.
- `UPDATE`.
- `DELETE`.
- `JOIN` basico.
- `GROUP BY` basico.

Python:

- Mantener proyectos personales.
- No abandonar la practica, pero dejar Java como prioridad de DAM.

## Objetivo concreto para septiembre

Llegar siendo capaz de crear por cuenta propia un programa sencillo en Java con:

- Menu de consola.
- Clases.
- Lista de objetos.
- Altas, consultas y pequenas modificaciones.

Ejemplos:

- Agenda de contactos.
- Gestor de productos.
- Inventario simple.
- Sistema basico de alumnos y notas.

## Que no estudiar todavia

Evitar por ahora:

- Spring Boot.
- Docker.
- React.
- Node.js.
- Kubernetes.
- IA avanzada.
- Microservicios.
- Patrones de diseno complejos.
- Arquitecturas complejas.

Motivo:

Meter demasiadas tecnologias antes de consolidar fundamentos crea conocimiento superficial y confusion.

## Criterio del profesor

El profesor debe tener este plan en cuenta durante las sesiones.

Si Enric intenta dispersarse con muchas tecnologias, se debe reconducir hacia:

1. Cerrar la calculadora V1.
2. Consolidar fundamentos.
3. Introducir Java de forma progresiva.
4. Introducir SQL en agosto.

La profundidad importa mas que avanzar rapido.
